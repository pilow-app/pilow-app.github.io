# 欢迎使用 Pilow

# Pilow 是 Planning in Flow （流式计划）的简称。

其的功能如其名，是通过流式日志完成项目任务和里程碑管理的工具。

Pilow 与「枕头」英文谐音，意味着，省下时间可以睡个懒觉。

# 起源

Pilow 是其开发者在日常的工作和项目管理中进行日志、任务和里程碑管理的工具，它的主要目标包括：

  • 提供进行项目的工作日志记录界面
  • 能登记待办任务、显示待办任务的「上下文」以及「解决状态」
  • 提供集中式的搜索结果
  • 可关联当前任务项目和里程碑的状态
  • 提供更自由或非正式的任务记录管理环境
  • 不依赖第三方服务，实现文档数据安全与隐私保护

# 现有工具问题

Pilow 的开发者在日常的工作和项目管理中，发现很难找到合适的工具同时满足以上的功能需求。以下是一些可能会存在的问题。

# 1. 文本编辑器

优点： 撰写自由，可按时间记录顺序记录。

缺点：

  • 不必要的手工编辑和排版操作：纯文本编辑器样式不够直观，排版容易混乱，另外部分文字处理较为麻烦，比如 Markdown 格式的待办任务需要将- [ ]手工处理成 - [x]；
  • 搜索结果不能集中显示，而只能逐条跳转显示；
  • 待办管理能力弱：Markdown 编辑器无法将未处理的待办任务集中显示，使用者只能全文逐步回溯查看未完成项目，有些编辑器不支持搜索待办状态。

# 2. 任务管理工具

优点： 可以灵活管理待办任务状态，一般可以搜索。

缺点：

  • 日志能力较弱：待办任务虽然可以按时间排序，但已完成任务通常和未完成任务分开，因此任务管理工具通常不反应项目事件时间顺序，用于做项目工作日志较为不便；
  • 显示过于简要：受限于任务管理的界面，通常不能显示过长的标题，因此导致其待办标题需要人为压缩或省略显示，或由应用提供额外的详情界面，导致任务的「上下文」不够直观，使用者容易对于长期未解决的任务遗忘其任务背景。

# 3. 大型项目管理工具

优点： 完整成体系的正式项目管理工具。

缺点：

  • 过于正式：大型项目工具常对事件录入由一定的标准和要求，导致常规项目管理中的日常事项和细节待办不可能记录在其中，因而常无法自由使用；
  • 侧重于项目阶段管理而不是项目的工作日志的记录；
  • 商业软件为主，需要位于授权环境使用，整体使用体验感较重。

# 4. 其他问题

  • 无法同时进行项目工作日志、待办任务管理、里程碑状态管理；
  • 数据可能需要上传至第三方服务，文档隐私和数据安全存在不确定性。

# 结语

如果你也有类似的感受，不妨来试试 #Pilow 吧！
