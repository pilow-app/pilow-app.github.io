# 快速开始

如果你不想花太多时间阅读文档，可以从这里快速开始

# 前置条件

作为前置条件，你需要使用现代浏览器，请下载 Chrome、Safari、Edge 或 Firefox 浏览器的最新版本或其他具备较新内核的浏览器。 因为运用了一些新的 Web 特性，因此 Pilow 无法支持 IE。

# 新建或打开本地文档

使用本文档上方的「现在开始」链接进入 Pilow 主界面，你可以选择「进入空文档」或「打开一个本地文档」。 如果你第一次使用，我们建议选择「进入空文档」，进行适当编辑后使用快捷键 Command/Ctrl + S 保存文件至本地。

你可以像普通文本编辑器一样使用 Pilow，Pilow不会默认自动保存文档，因此请定期手工保存。保存后文档将 Markdown（.md) 文件形式存在。每次保存都将更新本地文件的内容。当你完成编辑确认保存（看到保存成功提示）后，可以关闭浏览器。下次需要继续时，只需要在首页选择「打开一个本地文档」即可恢复工作。

# 添加和删除 Block

Block 在 Pilow 中是一个编辑单位，直观地说，当你新建或打开文档后，时间线上每个日期时间字符串之间的部分即为一个 Block，其上方的日期时间即为该Block 对应的创建时间。

一个Block的语义范围是由使用者决定的，一般会包括一天内关联性较强的事项。一天可以创建多个Block，如果没有太多事项，也可以创建一个 Block 或者不创建。

作为试用，现在请通过点击左上方「添加」连续创建 10 个 Block。

点击 Block 上方日期右侧的 即可以删除该 Block。

注意：Pilow 的筛选功能是以 Block 为单位的，使用者需要考虑到该因素。

# 插入待办任务

  1. 在刚刚创建的 Block 中，随便找一个或几个，插入待办事项。

  2. 插入待办事项可以通过使用Ctrl+Shift+7 也可以在行首输入英文 / 选择「Todo List」。

  3. 找另一个 Block， 重复 2 中的操作，但将其左方的完成状态勾选，此时你会看到一个蓝色☑️。

  4. 点击左上方的「待办」按钮，可以查看所有包含 没有完成的 待办任务的 Block。

Voilà！现在你就可以在一个界面看所有没有完成的事项，而且可以看到其关联的事项完成的状态（同一个Block中的其他任务），以及任务的上下文（同一个Block中的其他文字描述）。

同时，你也可以修改这些被筛选出的 Block 中的任务状态（在左边白框勾选）

# 使用「全部」

在查看完待办任务后，点击左上角这个平平无奇的「全部」，你将回到文档主页面视图，这里你可以看到文档所有的 Block。该按键也默认将窗口移动到文档的末端，便于你查看最新的文档部分和增加新的内容。

当你向上滚动文档查看历史 Block 时，如果想立刻滚动回至文档底部，也可以点击「全部」。

# 使用「里程碑式图」

在其中一个 Block 输入以下内容

#里程碑1-项目采购阶段-10%

紧接着在下一个Block 输入

已和供应商进行了邀标范围沟通 #里程碑1.1-招投标-30%

此时点击左下方「Pilow」主图标，点击「里程碑式图」（或者使用快捷键Command/Ctrl+M），这时你可以初步体验到里程碑更新的功能。更多详细说明请参考里程碑功能模块详细介绍。

# 再次「保存」

进行适当编辑后，仍然请再次记录使用快捷键 Command/Ctrl + S 保存文件至本地。然后你可以去本地看看是不是真的保存在了最新版本。

值得注意的是，你在使用 Pilow 的整个过程中，该文档的数据处理均在本地完成，相对于第三方商业服务，数据更安全，隐私更有保障，你也可以更放心！
