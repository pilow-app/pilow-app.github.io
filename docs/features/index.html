# Pilow 特性

# 1. 日志流 + 待办任务 + 里程碑 三合一

借助 Markdown 格式，你就可以轻松优美地进行项目工作日志记录，同时按照自然的使用习惯记录工作日志流中的待办任务。借助 #hashtag，更可以轻松进行关键字搜索和里程碑登记。

而这一切都在一个最简单 Markdown 文本中进行，而且这个文本文件保存在你的本地磁盘上，所有的数据处理均在本地进行。

Pilow 创造性使用 Markdown 文档，让 日志流 + 待办任务 + 里程碑 三合一。

Pilow 非常轻量，适合项目和工作中的非正式任务和进度管理。

# 2. 全面支持 MarkDown

Pilow 使用 MarkDown 作为数据存储方式，它本质只是一个纯文本文件，这意味着你可以随身携带，同时以任何其他文本编辑器打开。

即使以后不想使用 Pilow，你仍然对自己的数据有 100% 的把控权。

# 3. 分区块的日志，各单元独立编辑、整体过滤

Pilow 尽管看上去像一个大 Word 文档，但你可以分区（Block）块进行编辑。

每个区块是独立的，也就意味着，当你在一个区块「撤销」和「重做」历史操作时，不需要将其他区块的修改内容重置。

举例，假设有三个段落：

A

B

C

当你顺次编辑 A、B、C后，在Word 中如果你想撤销 A 的编辑，你需要一并撤销C、B 中的编辑。

在Pilow 中，如果将 A、B、C 分在不同区块（Block）中，你将只需将鼠标移动到 A ，然后使用 Command/Ctrl+Z 即可， 而B、C不受影响。

# 4. 聚合呈现待办任务、更新待办状态

Pilow 解决了传统纯文本中无法聚合待办 & 无法集中更新待办任务的问题。

# 5. 支持 #hashtag 标签 快速过滤

你可以在 Markdown 中加一个标签，可能是一个重要的需要持续跟踪的事项，或者一个里程碑。下次你看到这个标签时，只需要点击它，就可以快速过滤出所有相关的内容部分。

# 6. 支持随日志登记里程碑标志，形成里程碑报告

只要在今日事项后接一个 #里程碑标签，你就可以随手登记里程碑状态了。

Pilow 会自动生成一个里程碑视图（表格），包含里程碑完成进度，和该里程碑相关的最新动态事项和对应时间。

# 7. 安全可控的本地化数据处理、存储。

Pilow 数据处理和运算在本地进行，数据以 MarkDown 形式存在本地磁盘，全程数据安全可控、隐私更有保障。

Markdown 文件便携轻量，在任何其他的电脑，只需要访问 Pilow 网站，打开你的文档，一切又可以继续！
